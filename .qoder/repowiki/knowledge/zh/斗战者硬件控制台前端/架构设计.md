- **入口与布局**：`main.jsx` 引导应用并注入 `ToastProvider`；`App.jsx` 采用侧边栏导航与主内容区的双栏布局，通过 `activeTab` 状态管理视图切换。
- **状态管理**：核心逻辑封装于 `useControlState` Hook，统一管理主题、遥测数据、UXTU 参数及系统设置，并利用 `localStorage` 实现持久化。
- **组件架构**：UI 分为基础原子组件（`ui/` 目录下的 Card, Gauge 等）与业务面板（`panels/` 目录下的 PerformancePanel 等）；`SortableDashboard` 集成 `@dnd-kit` 实现仪表板卡片的拖拽排序与显隐控制。
- **通信层**：`uxtuAdapter.js` 封装了与后端 C# HAL 的交互，包括 REST API 调用（如 `/api/uxtu/apply`）及 WebSocket 实时遥测流（`ws://127.0.0.1:3100/ws`）。
- **降级策略**：当后端 WebSocket 不可用时，`useControlState` 自动切换至基于 `mockTelemetry` 的本地模拟数据生成逻辑，确保 UI 可用性。
