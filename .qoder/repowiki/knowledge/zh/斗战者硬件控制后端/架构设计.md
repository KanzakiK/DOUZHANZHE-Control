- **分层架构**：采用 `Douzhanzhe.API` (表现层/入口) 与 `Douzhanzhe.HAL` (硬件抽象层) 分离的设计。API 层通过依赖注入管理 HAL 单例，负责路由分发与状态持久化；HAL 层封装底层驱动调用。
- **硬件抽象策略**：`HardwareAbstractionLayer` 统一协调多种硬件访问路径：通过 `DriverBridge` (基于 `inpoutx64.dll`) 进行 EC 物理内存映射与 IO 端口操作；通过 `SmuController` 调用外部 `ryzenadj.exe` 子进程实现 AMD CPU 调优；通过 `WmiInterface` 利用 ACPI WMI 方法控制风扇与 GPU 模式。
- **通信机制**：支持 HTTP RESTful API 用于常规控制与配置读写，以及 WebSocket (`/ws`) 配合 `TelemetryBackgroundService` 实现高频遥测数据（温度、频率、转速）的实时推送。
- **前端集成**：内置静态文件服务 (`wwwroot`) 并配置 `MapFallbackToFile`，实现前后端一体化部署。
