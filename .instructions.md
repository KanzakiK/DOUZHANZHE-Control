## 当前状态 (已实现)

- **实时遥测**: CPU/GPU 温度、占用率、频率、风扇 RPM、内存/磁盘
- **硬件访问**: EC 寄存器 0x1C(温度)、0x9D/0x9E(CPU风扇)、0x96/0x97(GPU风扇)
- **SMU 控制**: 通过 RyzenAdj 写入 TDP/温度墙 (Dragon Range 写入后崩溃是已知问题)
- **5 性能模式**: 安静(35W)/均衡(55W)/游戏(85W)/狂暴(120W)/自定义
- **5 套主题**: 赛博霓虹/极简现代/极客数据/机甲紫黑
- **WebSocket**: 实时数据推送，断线自动 mock 降级
- **风扇最大值**: CPU=4400 RPM, GPU=8200 RPM
- **Git**: 已初始化并首次提交 (46 文件, 6626 行)

## 已知问题

1. **JSON 元数据假阳性** — 工具输出末尾的 `` 是 VS Code Copilot 基础设施缓存元数据，**不是文件内容**。通过 `node -e "fs.readFileSync(...)"` 字节读取验证文件是干净的。不要试图"修复"不存在的污染。
2. **非管理员模式**: 风扇 EC 读取需管理员权限，当前以非管理员运行
3. **Dragon Range SMU**: RyzenAdj 写入后崩溃 (ACCESS_VIOLATION 0xC0000005) 是已知行为，写入已成功
4. **空目录残留**: 迁移后旧 `douzhanzhe-console\` 空目录被 VS Code 锁定，关闭 VS Code 后删除

## 技术要点

- **前端端口**: 5173 (Vite)
- **后端端口**: 3099 (Express + WS)
- **EC 读取**: WinRing0x64.dll → 端口 0x66/0x62
- **GPU 数据**: nvidia-smi
- **CPU 频率**: WMI PercentProcessorPerformance
- **内存频率**: WMI Win32_PhysicalMemory.Speed
- **构建**: `npm run build` → `dist/`
- **服务器依赖**: `cd server && npm install` (系统信息、ws 等)

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/telemetry | 遥测数据 JSON |
| GET | /api/ryzenadj/info | SMU 参数信息 |
| POST | /api/uxtu/apply | 应用 SMU 限制参数 |
| WS  | / | 实时遥测推送 (2s 间隔) |
